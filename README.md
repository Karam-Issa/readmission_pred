# 30-Day Hospital Readmission Predictor

**Karam Issa | kissa3@gatech.edu**

A machine learning pipeline that predicts whether a hospital patient will be readmitted within 30 days of discharge. Built on Synthea-generated FHIR R4 data using XGBoost, with a FastAPI REST endpoint for serving predictions.

---

## Project Summary

Hospital readmissions cost the US healthcare system an estimated $26 billion annually. The Centers for Medicare & Medicaid Services (CMS) penalizes hospitals with excess readmission rates through the Hospital Readmissions Reduction Program (HRRP). This project builds a binary classifier that flags high-risk patients at discharge, enabling proactive intervention.

**Best model:** XGBoost — ROC-AUC = **0.9053**

## Quickstart

### 1. Clone and set up environment

```bash
git clone https://github.com/YOUR_USERNAME/readmission-predictor.git
cd readmission-predictor
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Generate data with Synthea

Download the Synthea jar from:
```
https://github.com/synthetichealth/synthea/releases/download/master-branch-latest/synthea-with-dependencies.jar
```

Place it in `data/raw/` then run:

```bash
cd data/raw
java -jar synthea-with-dependencies.jar \
  -p 10000 \
  -s 42 \
  --exporter.years_of_history=0 \
  --exporter.hospital.fhir.export=false \
  Massachusetts
```

### 3. Run the pipeline

```bash
# Parse FHIR JSON → feature matrix
python src/fhir_parser.py \
  --input data/raw/output/fhir/ \
  --output data/processed/features.csv

# Preprocess → train/test splits
python src/preprocess.py

# Train baseline model
python src/train_baseline.py

# Train advanced models (Random Forest + XGBoost)
python src/train_advanced.py

# SHAP explainability
python src/shap_analysis.py
```

### 4. Start the API

```bash
uvicorn api.main:app --reload
```

API runs at **http://127.0.0.1:8000**
Interactive docs at **http://127.0.0.1:8000/docs**

### 5. Run tests

```bash
pytest tests/test_pipeline.py -v
```

## API Usage

### `POST /predict`

Accepts patient data at discharge and returns a readmission risk score.

**Request:**
```json
{
  "age": 64.7,
  "gender_male": 1,
  "length_of_stay_days": 5.54,
  "prior_admissions": 12,
  "num_conditions": 25,
  "num_medications": 50,
  "num_procedures": 11,
  "has_prior_ed_visit": 1,
  "is_inpatient": 1
}
```

**Response:**
```json
{
  "readmission_risk": 0.5439,
  "risk_label": "Medium Risk",
  "threshold": 0.5,
  "model": "XGBClassifier"
}
```

**Risk labels:**
| Label | Probability |
|---|---|
| Low Risk | < 0.30 |
| Medium Risk | 0.30 – 0.60 |
| High Risk | > 0.60 |

## Feature Importance (SHAP)

| Rank | Feature | Description |
|---|---|---|
| 1 | num_medications | Active medications at admission |
| 2 | num_procedures | Procedures during encounter |
| 3 | num_conditions | Active diagnoses at admission |
| 4 | length_of_stay_days | Admission duration |
| 5 | prior_admissions | Prior inpatient visits |
| 6 | has_prior_ed_visit | ED visit in prior 6 months |
| 7 | age | Patient age at admission |
| 8 | gender_male | Patient gender |
| 9 | is_inpatient | Encounter type |

---

## Limitations

- **Synthetic data:** Generated by Synthea, not real clinical records. Medication and procedure counts are inflated due to lifetime accumulation — in real EHR data, `prior_admissions` and `age` would likely rank higher as predictors.
- **No cloud deployment:** Model runs locally. No SMART on FHIR integration.
- **Static model:** Trained offline on a fixed dataset. No real-time data ingestion.
- **Demo dataset:** 10,000 patients from Massachusetts only.
